# Issue Tracker

Issues for this repo are reported directly in conversation by the user — there is no external tracker (no GitHub Issues, Jira, Linear, etc.).

## Workflow

- When the user describes a bug or feature request, treat it as an incoming issue.
- Use the triage skill to evaluate and track its state internally.
- There is no CLI command to run, no API to call, and no files to write for issue creation.
- If a decision or resolution is worth persisting, summarise it in the conversation or ask the user if they'd like it noted somewhere.
